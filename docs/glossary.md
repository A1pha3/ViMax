## 核心概念

### Agent
- **中文**: 智能体
- **定义**: 系统中执行特定任务的独立模块，每个智能体负责特定的功能
- **示例**: Screenwriter（编剧智能体）、StoryboardArtist（分镜师智能体）
- **使用场景**: "CharacterExtractor 智能体负责从剧本中提取角色信息"

### Pipeline
- **中文**: 流水线
- **定义**: 完整的视频生成工作流程，协调多个智能体按顺序或并行工作
- **示例**: Idea2Video Pipeline、Script2Video Pipeline、Novel2Video Pipeline
- **使用场景**: "Idea2Video 流水线将简单创意扩展为完整视频"

### Multi-Agent System
- **中文**: 多智能体系统
- **定义**: 由多个自主智能体组成的系统，通过协作完成复杂任务
- **示例**: ViMax 的整体架构
- **使用场景**: "ViMax 采用多智能体系统架构，模拟真实的影视制作团队"

### Character Portrait
- **中文**: 角色画像
- **定义**: 为角色生成的标准形象图像，用于保持角色在不同场景中的一致性
- **示例**: 角色的前视图、侧视图、后视图
- **使用场景**: "生成角色画像以确保角色在整个视频中的外观一致"

### Reference Image
- **中文**: 参考图 / 参考图像
- **定义**: 用于指导图像或视频生成的参考图片，帮助保持视觉一致性
- **示例**: 角色画像、之前生成的关键帧
- **使用场景**: "使用角色画像作为参考图像生成新的镜头"

### Storyboard
- **中文**: 分镜 / 分镜脚本
- **定义**: 将剧本转化为视觉化的镜头序列，描述每个镜头的构图和内容
- **示例**: 包含镜头描述、摄像机角度、运动方式的文档
- **使用场景**: "StoryboardArtist 将剧本转化为详细的分镜脚本"


### Shot
- **中文**: 镜头
- **定义**: 视频中的一个连续片段，从一个摄像机位置拍摄的内容
- **示例**: 特写镜头、远景镜头、跟踪镜头
- **使用场景**: "每个场景包含 5-10 个镜头"

### Frame
- **中文**: 帧 / 关键帧
- **定义**: 视频中的单个静态图像，关键帧是镜头中的重要帧
- **示例**: 首帧（first frame）、末帧（last frame）
- **使用场景**: "为每个镜头生成首帧和末帧"

### Scene
- **中文**: 场景
- **定义**: 发生在同一时间和地点的一系列镜头的集合
- **示例**: "公园场景"、"办公室场景"
- **使用场景**: "剧本被分解为 3 个场景"

### Camera
- **中文**: 摄像机 / 机位
- **定义**: 拍摄视角和位置，相同摄像机的镜头共享相似的视角
- **示例**: 摄像机索引 0、摄像机索引 1
- **使用场景**: "使用相同摄像机的镜头会被分组处理"

### Camera Tree
- **中文**: 摄像机树
- **定义**: 组织镜头之间连续性关系的树形结构，用于优化生成顺序
- **示例**: 父摄像机和子摄像机的层级关系
- **使用场景**: "构建摄像机树以提高角色和场景的一致性"

## 系统架构

### Layer
- **中文**: 层 / 架构层
- **定义**: 系统架构中的功能分层
- **示例**: 输入层、流水线层、智能体层、工具层、输出层
- **使用场景**: "ViMax 采用五层架构设计"

### Input Layer
- **中文**: 输入层
- **定义**: 接收和验证用户输入的系统层
- **示例**: 接收创意、剧本、配置文件
- **使用场景**: "输入层负责验证 API Key 和配置参数"

### Pipeline Layer
- **中文**: 流水线层
- **定义**: 协调整体工作流程的系统层
- **示例**: Idea2Video、Script2Video 流水线
- **使用场景**: "流水线层是系统的核心调度中心"

### Agent Layer
- **中文**: 智能体层
- **定义**: 执行具体任务的智能体所在的系统层
- **示例**: 包含所有智能体的实现
- **使用场景**: "智能体层包含 13 个专门的智能体"

### Tool Layer
- **中文**: 工具层
- **定义**: 封装外部 AI 服务调用的系统层
- **示例**: 图像生成工具、视频生成工具
- **使用场景**: "工具层提供统一的接口访问不同的 AI 模型"

### Output Layer
- **中文**: 输出层
- **定义**: 组织和保存生成产物的系统层
- **示例**: 视频文件、图像文件、JSON 数据
- **使用场景**: "输出层负责保存所有中间文件和最终视频"

## 智能体相关

### Screenwriter
- **中文**: 编剧 / 编剧智能体
- **定义**: 负责创作和扩写剧本的智能体
- **功能**: 将创意扩展为故事，编写结构化剧本
- **使用场景**: "Screenwriter 将一句话创意扩展为完整的三幕剧本"

### Character Extractor
- **中文**: 角色提取器 / 角色分析师
- **定义**: 从剧本或小说中提取角色信息的智能体
- **功能**: 识别角色、提取静态和动态特征
- **使用场景**: "CharacterExtractor 从剧本中识别出 5 个主要角色"

### Storyboard Artist
- **中文**: 分镜师 / 分镜设计师
- **定义**: 将剧本转化为视觉分镜的智能体
- **功能**: 设计镜头序列、确定构图和角度
- **使用场景**: "StoryboardArtist 为场景设计了 8 个镜头"

### Camera Image Generator
- **中文**: 摄影师 / 图像生成器
- **定义**: 生成关键帧图像的智能体
- **功能**: 构建摄像机树、生成转场视频、生成关键帧
- **使用场景**: "CameraImageGenerator 为每个镜头生成首帧图像"

### Character Portraits Generator
- **中文**: 造型师 / 角色画像生成器
- **定义**: 生成角色标准画像的智能体
- **功能**: 生成角色的前视图、侧视图、后视图
- **使用场景**: "CharacterPortraitsGenerator 为每个角色生成三视图画像"

### Reference Image Selector
- **中文**: 参考图选择器 / 资料管理员
- **定义**: 选择最合适参考图像的智能体
- **功能**: 从可用图像中智能选择参考图
- **使用场景**: "ReferenceImageSelector 选择最相关的角色画像作为参考"

### Best Image Selector
- **中文**: 选片导演 / 最佳图像选择器
- **定义**: 从候选图像中选择最佳图像的智能体
- **功能**: 评估图像质量和一致性
- **使用场景**: "BestImageSelector 从 3 个候选图像中选出最佳的一张"

### Global Information Planner
- **中文**: 统筹 / 场记 / 全局信息规划器
- **定义**: 维护全局角色一致性的智能体
- **功能**: 跨场景管理角色信息
- **使用场景**: "GlobalInformationPlanner 确保角色在不同场景中的特征一致"

### Scene Extractor
- **中文**: 场景提取器 / 场景分析师
- **定义**: 从事件描述中提取场景的智能体
- **功能**: 将事件转化为具体场景剧本
- **使用场景**: "SceneExtractor 从事件中提取出 4 个场景"

### Event Extractor
- **中文**: 事件提取器 / 情节分析师
- **定义**: 从小说中提取关键事件的智能体
- **功能**: 识别关键情节节点
- **使用场景**: "EventExtractor 从小说中提取了 12 个关键事件"

### Novel Compressor
- **中文**: 小说压缩器 / 小说压缩师
- **定义**: 压缩长篇小说的智能体
- **功能**: 保留核心情节，去除冗余描述
- **使用场景**: "NovelCompressor 将 10 万字小说压缩到 3 万字"

## 流水线相关

### Idea2Video
- **中文**: 创意到视频 / 创意生成视频
- **定义**: 将简短创意扩展为完整视频的流水线
- **输入**: 1-3 句话的创意描述
- **输出**: 完整视频
- **使用场景**: "使用 Idea2Video 流水线快速验证创意"

### Script2Video
- **中文**: 剧本到视频 / 剧本生成视频
- **定义**: 将完整剧本转换为视频的流水线
- **输入**: 标准格式的剧本
- **输出**: 完整视频
- **使用场景**: "Script2Video 流水线适合影视预演"

### Novel2Video
- **中文**: 小说到视频 / 小说改编视频
- **定义**: 将长篇小说改编为系列视频的流水线（实验性）
- **输入**: 长篇小说文本
- **输出**: 系列视频
- **使用场景**: "Novel2Video 流水线用于文学作品改编"

### Working Directory
- **中文**: 工作目录
- **定义**: 存储中间文件和输出结果的目录
- **示例**: `.working_dir/idea2video`
- **使用场景**: "所有生成的文件保存在工作目录中"

### Configuration File
- **中文**: 配置文件
- **定义**: YAML 格式的系统配置文件
- **示例**: `configs/idea2video.yaml`
- **使用场景**: "在配置文件中设置 API Key 和模型选择"

## 视频制作术语

### Slugline
- **中文**: 场景标题 / 场景标识
- **定义**: 剧本中标识场景的标题行
- **格式**: `INT./EXT. 地点 - 时间`
- **示例**: `EXT. PARK - DAY`
- **使用场景**: "每个场景以 slugline 开始"

### Action
- **中文**: 动作描述 / 场景描述
- **定义**: 剧本中描述场景中发生的事情
- **示例**: "John walks into the room."
- **使用场景**: "动作描述应该简洁且易于视觉化"

### Dialogue
- **中文**: 对话
- **定义**: 剧本中角色说的话
- **格式**: `角色名: 对话内容`
- **示例**: `JOHN: Hello, how are you?`
- **使用场景**: "对话应该符合角色性格"

### Close-up
- **中文**: 特写 / 特写镜头
- **定义**: 近距离拍摄主体的镜头
- **示例**: 人物面部特写
- **使用场景**: "使用特写镜头强调角色的情绪"

### Wide Shot
- **中文**: 远景 / 全景镜头
- **定义**: 展示整个场景的镜头
- **示例**: 展示整个房间或户外环境
- **使用场景**: "远景镜头用于建立场景环境"

### Medium Shot
- **中文**: 中景 / 中景镜头
- **定义**: 拍摄主体部分的镜头
- **示例**: 人物腰部以上
- **使用场景**: "中景镜头是最常用的镜头类型"

### Tracking Shot
- **中文**: 跟踪镜头 / 移动镜头
- **定义**: 摄像机跟随主体移动的镜头
- **示例**: 跟随角色行走
- **使用场景**: "跟踪镜头增加动态感"

### Transition
- **中文**: 转场 / 过渡
- **定义**: 从一个镜头切换到另一个镜头的方式
- **示例**: 淡入淡出、切换、推拉
- **使用场景**: "平滑的转场使视频更流畅"


### First Frame (FF)
- **中文**: 首帧
- **定义**: 镜头的第一帧图像
- **用途**: 作为视频生成的起始参考
- **使用场景**: "生成首帧作为镜头的关键帧"

### Last Frame (LF)
- **中文**: 末帧
- **定义**: 镜头的最后一帧图像
- **用途**: 用于变化较大的镜头，确保运动的终点
- **使用场景**: "对于变化较大的镜头，同时生成首帧和末帧"

### Keyframe
- **中文**: 关键帧
- **定义**: 视频中的重要帧，用于定义运动的起点和终点
- **示例**: 首帧和末帧都是关键帧
- **使用场景**: "基于关键帧生成中间的运动过渡"

### Aspect Ratio
- **中文**: 宽高比
- **定义**: 图像或视频的宽度与高度的比例
- **示例**: 16:9、9:16、1:1
- **使用场景**: "设置视频的宽高比为 16:9"

### Resolution
- **中文**: 分辨率
- **定义**: 图像或视频的像素尺寸
- **示例**: 480p、720p、1080p、4K
- **使用场景**: "生成 1080p 高清视频"

### Frame Rate (FPS)
- **中文**: 帧率
- **定义**: 每秒显示的帧数
- **示例**: 24 fps、30 fps、60 fps
- **使用场景**: "电影通常使用 24 fps 的帧率"

## 技术术语

### Asynchronous / Async
- **中文**: 异步
- **定义**: 不阻塞主线程的并发执行方式
- **技术**: Python asyncio
- **使用场景**: "使用异步方式并行生成多个镜头"

### Parallel Processing
- **中文**: 并行处理
- **定义**: 同时执行多个任务以提高效率
- **示例**: 同时生成多个角色画像
- **使用场景**: "并行处理可以显著减少总体生成时间"

### Retry Mechanism
- **中文**: 重试机制
- **定义**: 失败后自动重新尝试的机制
- **实现**: 使用 tenacity 库
- **使用场景**: "API 调用失败时自动重试最多 3 次"

### Caching
- **中文**: 缓存
- **定义**: 存储已计算结果以避免重复计算
- **示例**: 缓存角色画像、中间结果
- **使用场景**: "使用缓存机制避免重复生成相同内容"

### Prompt
- **中文**: 提示词
- **定义**: 用于指导 AI 模型生成内容的文本描述
- **示例**: "一只可爱的猫咪，卡通风格"
- **使用场景**: "优化提示词可以提高生成质量"

### Prompt Engineering
- **中文**: 提示词工程
- **定义**: 设计和优化提示词的技术
- **目的**: 提高 AI 生成质量
- **使用场景**: "通过提示词工程确保输出格式符合要求"

### LLM (Large Language Model)
- **中文**: 大语言模型
- **定义**: 基于深度学习的大规模语言理解和生成模型
- **示例**: GPT-4、Gemini、Claude
- **使用场景**: "使用 LLM 驱动智能体进行推理和生成"

### MLLM (Multimodal Large Language Model)
- **中文**: 多模态大语言模型
- **定义**: 能够理解和生成多种模态（文本、图像、视频）的模型
- **示例**: GPT-4 Vision、Gemini Pro Vision
- **使用场景**: "使用 MLLM 评估图像质量和一致性"


### RAG (Retrieval-Augmented Generation)
- **中文**: 检索增强生成
- **定义**: 结合信息检索和文本生成的技术
- **用途**: 从知识库检索相关信息辅助生成
- **使用场景**: "Novel2Video 使用 RAG 检索原文细节"

### Vector Database
- **中文**: 向量数据库
- **定义**: 存储和检索向量嵌入的数据库
- **示例**: FAISS、Pinecone、Weaviate
- **使用场景**: "使用 FAISS 向量数据库进行语义检索"

### Embedding
- **中文**: 嵌入 / 向量嵌入
- **定义**: 将文本转换为数值向量的表示
- **用途**: 用于语义相似度计算
- **使用场景**: "将文本转换为嵌入向量进行相似度搜索"

### Reranker
- **中文**: 重排序器
- **定义**: 对检索结果进行重新排序的模型
- **示例**: BGE Reranker
- **使用场景**: "使用 Reranker 提高检索结果的相关性"

### Pydantic Model
- **中文**: Pydantic 模型
- **定义**: 用于数据验证和序列化的 Python 类
- **用途**: 确保数据结构的类型安全
- **使用场景**: "使用 Pydantic 模型定义智能体的输入输出"

### JSON Schema
- **中文**: JSON 模式
- **定义**: 定义 JSON 数据结构的规范
- **用途**: 验证和文档化数据格式
- **使用场景**: "使用 JSON Schema 确保配置文件格式正确"

### API Key
- **中文**: API 密钥
- **定义**: 用于身份验证的密钥字符串
- **安全**: 应妥善保管，不要泄露
- **使用场景**: "在配置文件中填入您的 API Key"

### Base URL
- **中文**: 基础 URL / API 端点
- **定义**: API 服务的基础地址
- **示例**: `https://api.openai.com/v1`
- **使用场景**: "设置 base_url 指向 API 服务地址"

### Endpoint
- **中文**: 端点 / API 端点
- **定义**: API 的具体访问地址
- **示例**: `/v1/chat/completions`
- **使用场景**: "调用图像生成端点"

## 工具与API

### Image Generator
- **中文**: 图像生成器 / 图像生成工具
- **定义**: 用于生成图像的 AI 工具
- **示例**: Google Imagen、DALL-E、Midjourney
- **使用场景**: "配置图像生成器用于生成关键帧"

### Video Generator
- **中文**: 视频生成器 / 视频生成工具
- **定义**: 用于生成视频的 AI 工具
- **示例**: Google Veo、Sora、Runway
- **使用场景**: "配置视频生成器用于生成镜头视频"

### Text-to-Video (T2V)
- **中文**: 文本生成视频
- **定义**: 根据文本描述生成视频
- **输入**: 文本提示词
- **输出**: 视频
- **使用场景**: "使用 T2V 模式从文本描述生成视频"

### Image-to-Video (I2V)
- **中文**: 图像生成视频
- **定义**: 根据图像和文本描述生成视频
- **输入**: 参考图像 + 文本提示词
- **输出**: 视频
- **使用场景**: "使用 I2V 模式从关键帧生成视频"

### First Frame to Video (FF2V)
- **中文**: 首帧生成视频
- **定义**: 基于首帧图像生成视频
- **输入**: 首帧图像 + 运动描述
- **输出**: 视频
- **使用场景**: "使用 FF2V 模式从首帧生成镜头视频"

### First-Last Frame to Video (FLF2V)
- **中文**: 首尾帧生成视频
- **定义**: 基于首帧和末帧生成中间过渡视频
- **输入**: 首帧 + 末帧 + 运动描述
- **输出**: 视频
- **使用场景**: "使用 FLF2V 模式确保镜头的起止状态"


### Google Veo
- **中文**: Google Veo（保持英文）
- **定义**: Google 开发的视频生成模型
- **版本**: Veo 2、Veo 3、Veo 3.1
- **特点**: 业界领先的视频生成质量
- **使用场景**: "使用 Google Veo 3.1 生成高质量视频"

### Google Gemini

## 流程术语

### Hierarchical Planning
- **中文**: 分层规划
- **定义**: 自顶向下逐步细化的规划策略
- **层次**: 创意 → 故事 → 场景 → 镜头 → 帧
- **使用场景**: "ViMax 采用分层规划策略处理复杂任务"

### Multi-Agent Collaboration
- **中文**: 多智能体协作
- **定义**: 多个智能体协同工作完成任务
- **模式**: 流水线模式、并行模式、迭代模式
- **使用场景**: "多智能体协作模拟真实的影视制作团队"

### Global Consistency
- **中文**: 全局一致性
- **定义**: 确保角色和场景在整个视频中保持一致
- **机制**: 角色画像、参考图选择、摄像机树
- **使用场景**: "全局一致性是长视频生成的关键挑战"

### Incremental Generation
- **中文**: 增量生成
- **定义**: 逐步生成内容，支持断点续传
- **优势**: 避免重复计算，节省时间和成本
- **使用场景**: "增量生成允许从失败点继续执行"

### Batch Processing
- **中文**: 批量处理
- **定义**: 一次处理多个任务
- **优势**: 提高效率，减少网络开销
- **使用场景**: "批量处理可以并行生成多个角色画像"

## 质量术语

### Variation Type
- **中文**: 变化类型 / 变化程度
- **定义**: 描述镜头变化幅度的分类
- **类型**: small（小）、medium（中）、large（大）
- **用途**: 决定是否需要生成末帧
- **使用场景**: "变化程度为 large 的镜头需要生成首帧和末帧"

### Consistency
- **中文**: 一致性
- **定义**: 角色、场景、风格在不同镜头中的统一性
- **类型**: 角色一致性、空间一致性、风格一致性
- **使用场景**: "保持角色一致性是视频生成的核心挑战"

### Quality
- **中文**: 质量
- **定义**: 生成内容的视觉质量
- **评估**: 清晰度、细节、准确性
- **使用场景**: "提高生成质量需要优化提示词和参数"

### Fidelity
- **中文**: 保真度
- **定义**: 生成内容与原始描述的匹配程度
- **评估**: 是否包含所有关键元素

## 使用说明

### 如何使用本术语表

1. **查找术语**: 使用目录快速定位术语类别
2. **理解定义**: 阅读中英文对照和详细定义
3. **查看示例**: 通过示例理解术语的实际应用
4. **保持一致**: 在文档中统一使用术语表中的翻译

### 术语翻译原则

1. **专有名词**: 保持英文原文（如 Google Veo、LangChain）
2. **技术术语**: 使用业界通用翻译（如 Agent → 智能体）
3. **新造词**: 选择最贴切的中文表达（如 Pipeline → 流水线）
4. **一致性**: 同一术语在所有文档中使用相同翻译
