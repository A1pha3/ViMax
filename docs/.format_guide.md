# ViMax 文档格式规范

> 本文档定义 ViMax 项目文档的统一格式标准，确保所有文档保持一致的风格和结构。

## 文件命名规范

- 使用小写字母和下划线：`getting_started.md`
- 避免使用驼峰命名：~~`GettingStarted.md`~~
- 避免使用连字符：~~`getting-started.md`~~

## 文档结构

### 标准文档模板

```markdown
# 文档标题

> 简短的一句话描述本文档的目的

## 目录

- [章节1](#章节1)
- [章节2](#章节2)

## 前置知识

在阅读本文档之前，建议您先了解：

- [相关文档](./related.md) - 简短说明
- 基本概念或技能

## 主要内容

### 章节1

内容...

### 章节2

内容...

## 相关资源

- [相关文档1](./doc1.md) - 说明
- [相关文档2](./doc2.md) - 说明

---

**最后更新**: YYYY-MM-DD
```

## 标题层级

### 一级标题 (#)
- 仅用于文档标题
- 每个文档只有一个一级标题

### 二级标题 (##)
- 用于主要章节
- 示例：`## 目录`、`## 前置知识`、`## 相关资源`

### 三级标题 (###)
- 用于子章节
- 示例：`### 步骤1`、`### 配置选项`

### 四级标题 (####)
- 谨慎使用，仅在必要时使用
- 避免过深的层级嵌套

## 代码块格式

### 带语言标识符

```python
# Python 代码示例
def hello():
    print("Hello, World!")
```

```yaml
# YAML 配置示例
key: value
```

```bash
# Bash 命令示例
ls -la
```

### 行内代码

使用反引号标记：`variable_name`、`function()`、`file.py`

## 列表格式

### 无序列表

- 使用连字符 `-` 而非星号 `*`
- 列表项首字母大写（中文无需大写）
- 列表项末尾不加句号（除非是完整句子）

示例：
```markdown
- 第一项
- 第二项
- 第三项
```

### 有序列表

1. 使用数字加点号
2. 保持缩进一致
3. 子列表使用两个空格缩进

示例：
```markdown
1. 第一步
  - 子步骤 A
  - 子步骤 B
2. 第二步
3. 第三步
```

## 表格格式

### 标准表格

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 值1 | 值2 | 值3 |
| 值4 | 值5 | 值6 |
```

### 对齐方式

```markdown
| 左对齐 | 居中 | 右对齐 |
|:-------|:----:|-------:|
| 内容   | 内容 | 内容   |
```

## 强调格式

### 粗体

- 使用双星号：`**重要内容**`
- 用于强调重要概念、术语
- 示例：**API Key**、**必需**

### 斜体

- 使用单星号：`*斜体文本*`
- 中文文档中较少使用
- 用于外文术语或特殊强调

### 代码格式

- 使用反引号：`` `code` ``
- 用于代码、命令、文件名、变量名
- 示例：`main.py`、`--help`、`api_key`

## 链接格式

### 内部链接

```markdown

[链接到章节](./target_file.md#章节名)
```

### 外部链接

```markdown
[链接文本](https://example.com)
```

### 参考链接

```markdown
详见 [系统架构](./architecture.md) 了解更多信息。
```

## 示例代码格式

### 完整示例

```python
# 文件名：example.py
# 说明：这是一个完整的示例

import asyncio

async def main():
    """主函数"""
    print("Hello, World!")

if __name__ == "__main__":
    asyncio.run(main())
```

### 代码片段

```python
# 仅展示关键部分
result = await pipeline(idea, user_requirement, style)
```

### 命令示例

```bash
# 安装依赖
uv sync

# 运行程序
python main.py
```

## 注释格式

### 代码注释

- Python：使用 `#` 注释
- YAML：使用 `#` 注释
- 注释应简洁明了

### 文档注释

- 使用 `<!-- 注释内容 -->` 进行 Markdown 注释
- 注释不会在渲染后显示

## 术语使用

### 统一术语

- 参考 [术语表](./glossary.md)
- 保持术语翻译一致
- 首次出现时可添加英文原文

示例：
```markdown
智能体 (Agent) 是系统中执行特定任务的独立模块。
```

### 专有名词

- 保持英文原文：Google Veo、LangChain
- 不需要翻译：API、JSON、YAML

## 空行规范

### 标题前后

- 标题前空一行
- 标题后不空行

### 段落之间

- 段落之间空一行

### 代码块前后

- 代码块前后各空一行

### 列表前后

- 列表前后各空一行

## 标点符号

### 中文标点

- 使用中文标点：，。！？：；
- 括号：（）而非 ()

### 英文标点

- 代码中使用英文标点
- 英文句子使用英文标点

### 混合使用

- 中文句子中的英文单词后使用中文标点
- 示例：使用 Python 编写代码。

## 文件结尾

- 文件末尾添加空行
- 添加更新日期（可选）

```markdown
---

**最后更新**: 2025-12-01
```

- [ ] 列表格式统一（使用 `-`）
